# Sales Messaging

## 一句话定位

Synapse 是面向企业 AI 系统的私有化执行平面，让模型能够执行代码，但不越过企业的安全边界。

## 三条主价值

### 1. 不出域

执行层部署在客户控制的环境中，适合对数据边界和内网控制有要求的团队。

### 2. 可审计

每次执行都带有 `request_id`、`tenant_id` 与审计记录，便于排查、复盘和合规沟通。

### 3. 不失控

资源限制、超时、配额与错误语义是产品接口的一部分，而不是演示时临时补上的保护层。

## 适用场景

- 企业内部 AI 代码助手
- PR Review Agent
- CI/CD 自动化中的受控代码执行

## 不建议的销售叙事

- 不强调“通用云沙箱平台”
- 不强调“比 E2B 更快”
- 不承诺浏览器、Desktop、在线 IDE
- 不承诺面向个人开发者的低价托管服务

## 常见客户问题

### 为什么不用模型厂商自带 code execution

当客户核心诉求是私有化、审计和边界控制时，托管执行通常不满足要求。

### 为什么不是一个完整 sandbox platform

Synapse v1 先聚焦企业 AI 执行层，把最关键的可控执行、审计与私有化交付打实。

### 典型成交前提

- 客户已有明确 agent 场景
- 客户已有内网或高合规要求
- 客户愿意投入一周左右完成 PoC 接入
