# Pricing Draft

## 定价方向

Synapse v1 建议采用企业私有化软件的定价方式，而不是低价按量 API 模式。

当前建议：

- 私有化年费许可证
- 按容量档定价
- 附带支持服务

## 容量维度

可用于报价分档的核心维度：

- 最大并发执行数
- 节点数量
- 支持环境数量
- 支持响应级别

## 报价结构建议

- Base License: 覆盖基础部署与固定容量
- Support: 标准支持或高级支持
- Services: PoC、上线辅导、定制集成

## 当前原则

- 不对外公布个人开发者价格
- 不以低价取代托管沙箱平台竞争
- 报价必须与客户部署边界、容量和支持义务绑定
